Project Scope

Project Description
The aim of this project is to create a website to showcase a software documentation of various IT projects.
Problem Benefits of Solving
User should see the list of project titles. Implementation of a list of projects on main page will allow user to get an overview of the website content.
User should be able to see projects documentation. A webpage with index referring to scope, requirements and interface documentation will allow user to clearly browse the webpage.
Additional information about some projects should be presented. User can get more detailed information about some projecs, such as bibliography or link to code documentation.
Indicative Schedule
Implementation Start Date: 22.07.2024 Implementation End Date: 01.08.2024
Solution Scope
During implementation the developer will:
  • Design the website.
  • Implement the webpages using html with css.
  • Prepare the documentation of projects to be displayable on website.
  • Deploy the website on GitHub Pages.
Project Assumptions
  1. Deplyment done by GitHub Pages.
  2. Continous development and maintenance after the end date in indicative schedule.
Key Risks Rating
Error on GitHub pages preventing users from visiting page. Low
Page growing too big over time to be deployed on GitHub pages. Medium
